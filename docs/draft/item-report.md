# Item Data Report — Pubkey Quest

Generated by the **item-reporter** agent (read-only audit of `game-data/items/*.json`).
This report describes the item catalog's schema, data-quality issues, cross-reference
integrity, sprite coverage, and balance notes — so a future **item-refiner** can
standardize it. No item JSON was edited.

**Snapshot:** 209 item files, 84 spells, 216 item sprite PNGs. All 209 items parse as
valid JSON; every `id` matches its filename.

---

## 0. How items are actually loaded (context for the findings)

- The JSON is **flat** — there is no nested `properties` object in any file. Type-specific
  fields (`damage`, `ac`, `provides`, `container_slots`, …) sit at the top level next to
  the base fields.
- The migration (`cmd/codex/migration/migration.go:516-544`) copies only
  `id, name, description, type, tags, rarity` into columns and **serializes the entire
  item JSON verbatim into a `properties` blob**. So `value`, `weight`, `image`, `img`,
  `damage`, etc. are all read out of that blob by key at runtime. **Nothing validates the
  key names** — a misspelled key (`substitues`, `restrain`) is simply invisible to whatever
  reads the correct key. This is why the schema drift below persists silently.
- `value` is the live worth field: the shop reads `item.Value` as `basePrice`
  (`cmd/server/api/game/shop.go:223`). The Go `types.Item` struct
  (`types/items.go:12`) still declares `Price int` and `Weight int` — this struct is
  **vestigial** for item loading (JSON has no `price`, and weight is fractional). Worth a
  cleanup but not load-bearing.
- `allowed_types` is read with a `switch` that already tolerates both a bare string and a
  list (`cmd/server/game/inventory/containers.go:122`), which is why the string-vs-list
  inconsistency (below) hasn't broken anything.

## 2. Schema by type

Base fields present on **all 209** items: `id, name, description, type, rarity, value,
weight, stack, tags, notes, image`. `img` (a category-placeholder, see §3) is present on
131. Below, **UNIV** = present on every item of that type; `n/total` = partial.

- **Simple/Martial Melee Weapons** (28): UNIV `damage, damage-type, gear_slot`.
  `range` + `range-long` on thrown melee only (Simple 5/10, Martial-melee 6/18; one melee
  has range but no range-long: `whip`-family — see §3).
- **Simple/Martial Ranged Weapons** (9): UNIV `damage, damage-type, ammunition, range,
  range-long`. `gear_slot` is UNIV on Simple but **4/5 on Martial Ranged** (one missing).
- **Light/Medium/Heavy Armor** (23): UNIV `ac, gear_slot`. `set` is partial:
  Light 6/6, Medium 6/8, Heavy 7/9 → **4 armor pieces have no `set`**
  (`breastplate`, `chain-shirt`, `shield`, `splint-greaves`). `shield`/`breastplate`/
  `chain-shirt` are legitimately setless standalones; `splint-greaves` is a real gap (§4).
- **Armor Set** (10): UNIV `contents` (list of `[piece-id, qty]`) + `set_bonus`
  (`{ac, effects[]}`). These are virtual bundles (weight 0).
- **Adventuring Gear** (64): the grab-bag. Containers within it carry `container_slots` +
  `allowed_types` (10 each); a few carry weapon fields (improvised weapons); one carries
  `effects_when_worn` (`backpack`). This type hosts most of the one-off keys (§3).
- **Pack** (7): UNIV `contents` (list of `[item-id, qty]`). No other type-specific fields.
- **Arcane Focus** (5): UNIV `gear_slot, provides`; `substitues` (**misspelled**) on 4/5.
- **Druidic Focus** (4): UNIV `gear_slot, provides, substitutes` (**correct spelling**).
- **Holy Symbol** (3): UNIV `gear_slot, provides`. No substitutes key at all.
- **Potion** (4): UNIV `heal, effects`. **Food** (2): UNIV `effects` (no `heal`).
- **Musical Instrument** (10): `performance` on **9/10** (one missing — §3).
- **Ammunition** (4): UNIV `gear_slot: "ammo"`.
- **Gaming Set** (4) / **Tools** (7): base fields only; 2 Tools carry improvised-weapon
  `damage`. No `gear_slot` on Gaming Sets/most Tools.

## 3. Data-quality issues (most valuable section)

### Confirmed known issues

1. **`substitues` typo** — the misspelled key `substitues` appears on **4 Arcane Focus
   items** (`crystal`, `orb`, `staff`, `wand`); the correct `substitutes` appears on **4
   Druidic Focus items** (`sprig-of-mistletoe`, `totem`, `wooden-staff`, `yew-wand`). Same
   concept, two spellings. *Extra wrinkle:* the 5th Arcane Focus (`rod`) has **neither**
   key, and Holy Symbols have neither. **Every one of these 8 values is an empty string
   `""`** — so the field is dead weight everywhere. **Fix:** pick `substitutes`, drop the
   typo; either populate them meaningfully or remove the key entirely.

2. **`img` vs `image`** — `image` is the canonical sprite path (universal, all 209).
   `img` is present on **131 items but is NOT a duplicate sprite path** — it is a
   **category-placeholder** like `"equipment/weapon.jpg"`, `"equipment/spell-component.jpg"`.
   All 131 `img` values disagree with `image` because they're a different thing (a legacy
   type-icon). **Fix:** drop `img` entirely — it's a vestigial per-category icon field that
   no longer matches the per-item sprite scheme.

3. **Naming-convention drift (hyphen vs underscore)** — `damage-type` (39), `range-long`
   (16) use hyphens, while `gear_slot`, `set_bonus`, `container_slots`, `allowed_types`,
   `effects_when_worn` use underscores. **Fix:** normalize to underscores
   (`damage_type`, `range_long`) — but confirm the reader keys first, since these are read
   by string.

4. **`damage-type` value drift** — the value vocabulary has a typo pair:
   `bludgeoning` (1: `mace`) vs `bludgeon` (9). Also two thrown weapons have
   `damage-type` blank/absent (`pick-miners`, `ram-portable`). Others: `slashing` 11,
   `piercing` 17, `fire` 1. **Fix:** standardize on `bludgeoning`.

5. **One-off / singleton keys** (all in Adventuring Gear):
   - **`pouch`** carries **two** redundant keys: `contains: "any"` (duplicates
     `allowed_types: "any"`) and `slots: "4"` (duplicates `container_slots: 20` — and the
     two even *disagree*, 4 vs 20). **Fix:** delete `contains` and `slots`.
   - **`quiver`** carries `container: true` — redundant with its `container` tag +
     `container_slots`. **Fix:** delete.
   - **`caltrops`** carries `restrain: "1d3"` (a bespoke restraint-damage key on one item).
     `net` expresses the same idea only via a `restraint` tag + a note. **Fix:** either
     formalize a restraint mechanic key across both, or drop `restrain`.
   - **`backpack`** carries `effects_when_worn` (the only item with it). Legit per the
     equip-effects system, but worth confirming it's wired.

6. **Type-expected field gaps:**
   - **Armor without `set`:** `splint-greaves` is listed in `splint-set.contents` but has
     **no `set` field pointing back** (asymmetric — see §4). (`breastplate`, `chain-shirt`,
     `shield` are legitimately standalone.)
   - **Musical Instrument without `performance`:** 1/10 (`bagpipe` — the only instrument
     with empty tags and no `performance`).
   - **Martial Ranged Weapon without `gear_slot`:** 1/5.

### New issues found

7. **63 items still say `"NEEDS DESCRIPTION"`** (30% of the catalog) — e.g. every basic
   weapon (`longsword`, `mace`, `shield`, `rapier`, …) plus `amulet`, `emblem`, `orb`,
   `rod`, `wand`, `sprig-of-mistletoe`, `totem`, `waterskin`. Big content gap.

8. **`net` is malformed as a weapon** — `type: "Martial Ranged Weapons"`, `damage: "0"`,
   `range-long: "320"` (every other weapon's range-long is single digits, so this is likely
   a units bug — feet vs grid squares), and `ammunition: "arrows"` (a net needs no ammo).
   **Fix:** model net as a thrown restraint, not an ammo weapon.

9. **`dart` mis-slotted** — `type: "Simple Ranged Weapons"` but `gear_slot: "ammo"` (it's a
   thrown weapon held in hand, not ammunition). Also references `ammunition: "arrows"`.

10. **`gear_slot` "hands" vs equipment model** — 33 items use `gear_slot: "hands"`, but the
    equipment struct (`types/items.go:26`) only has `mainhand`/`offhand` slots
    (`mainhand` appears on 12 items, `offhand` on 1). Confirm whether "hands" resolves to
    mainhand at equip time, or is an unmapped value.

11. **`gold-piece` stack `1000000000000`** — a 1e12 sentinel for "effectively unlimited".
    Harmless but worth a named constant / comment; other stackables use small caps.

## 4. Cross-reference integrity

### Focus `provides` → Spell Component  ✅ all valid
All 12 focus items with `provides` point to a real item that is a `Spell Component`
(amulet→blessed-incense, crystal→quartz-dust, orb→ether-essence, rod→sulfur,
staff→mana-crystals, wand→arcane-powder, totem→bone-dust, wooden-staff→bark-shavings,
yew-wand→tree-sap, sprig-of-mistletoe→pollen, emblem→sacred-oil, reliquary→holy-water).
No focus-type item is missing `provides`.

### Spell requirements vs components  (semantic gaps, not broken links)
- Every component **required by a spell** (`material_component.required[].component`)
  resolves to a real item — **0 broken refs**. 13 distinct components are used across
  spells (spirit-dust in 5, pollen in 3, blessed-incense/ash/mana-crystals in 2 each, etc.).
- **11 of the 24 Spell Components are never required by any spell** *and* never provided by
  a focus: `demon-ichor, dragon-scale, ether-essence, mushroom-spores, phoenix-feather,
  starlight-essence (legendary), void-crystal (legendary), quartz-dust, bone-dust,
  bark-shavings, holy-water`. Several of these are the very components some foci *provide*
  (quartz-dust, bone-dust, bark-shavings, holy-water, ether-essence) — so the foci grant
  components no spell asks for. **These are content gaps in the spell data, not item bugs**
  — flag for the spell-refiner: either wire spells to use them or trim unused components.

### Armor `set` ↔ Armor Set `contents`  — 1 asymmetric link
- All **19** armor pieces with a `set` point to a real Armor Set that lists them back. ✅
- **`splint-set.contents` lists `splint-greaves`, but `splint-greaves` has no `set` field**
  — one-directional. **Fix:** add `"set": "splint-set"` to `splint-greaves`.

### Weapon `ammunition` — 4 broken links (real bug)
`ammunition` is used as a free-text string and 4 weapons name a value that is **not a valid
ammo item id**:
- `crossbow-hand`, `crossbow-heavy`, `crossbow-light` → `"bolts"` (should be
  **`crossbow-bolts`**).
- `sling` → `"sling bullets"` (should be **`sling-bullet`**).

Additionally, several weapons point at the wrong-but-existing ammo:
- `dart` and `net` → `"arrows"` (neither uses arrows — `dart`/`net` are thrown).
Valid links: `blowgun→blowgun-needle`, `longbow`/`shortbow`→`arrows`. **Fix:** normalize
`ammunition` to the ammo item id (`crossbow-bolts`, `sling-bullet`) and drop it from
thrown weapons.

### Container `allowed_types` — schema inconsistent (tolerated, still worth fixing)
`allowed_types` is a **bare string** on 6 containers (`"any"` on backpack/barrel/chest/
case-map-and-scroll/pouch/sack) and a **list** on 3 (`["Food"]` basket,
`["Spell Component"]` component-pouch, `["Ammunition"]` quiver). One value —
`spellbook`'s `"spell-scroll"` — is **not a real item `type`** (no item has type
`spell-scroll`), so that container can never accept anything by type match. **Fix:**
make `allowed_types` always a list; replace `"any"` with an explicit empty/`["any"]`
convention; either add a `spell-scroll` type or point spellbook at the real type.

### Pack / Armor-Set `contents` — all refs valid ✅
Every `[id, qty]` member in all 7 Packs and 10 Armor Sets resolves to a real item.

## 6. Balance / semantic notes

### Value vs rarity outliers
- **37 weapons have `value: 0`** (every base weapon: `longsword`, `greatsword`,
  `battleaxe`, `dagger`, `longbow`, … plus `pouch`, `quiver`). Since the shop reads `value`
  as base price, **these are literally free to buy and worthless to sell** — the single
  biggest balance gap. Needs a full weapon-pricing pass (PHB gp values × the game's ×100
  copper-ish scale would be a starting point, given a `longsword`-tier item elsewhere runs
  in the thousands).
- **`case-map-and-scroll`: rarity `rare`, value `1`** — rarity/value contradiction (should
  be common).
- **`breastplate`: common, value 40000** while `halfplate-cuirass` (45000) is `rare` — the
  common/rare boundary is set inconsistently across armor. `spyglass` is `common` at
  100000 (the single priciest common item). `supreme-healing` common at 50000.
- Rare Spell Components span 500–10000; legendary ones (`starlight-essence` 10000,
  `void-crystal` 8000) — but note `starlight-essence` (legendary) is priced *above*
  `void-crystal` yet neither is used by any spell (§4).

### Rarity + tag vocabularies
- **Rarity** vocab is clean: `common/uncommon/rare/legendary` only.
- **Tags** vocab (26 distinct) is mostly consistent, but:
  - **59 items have an empty `tags: []`** — most of Adventuring Gear (`bedroll`, `rope`,
    `bucket`, all the Gaming Sets, most instruments, `waterskin`, …). Not wrong, but these
    items are unfindable by tag and get no `equipment`/`container`/etc. classification.
  - `spell_component` (underscore) vs `armor-set`/`light-source`/`oil-burning` (hyphen) —
    tag-naming isn't internally consistent either.
  - Good news: **all Potions/Food carry `consumable`**, **all focus-types carry `focus`**,
    and **all 24 Spell Components carry `spell_component`** — those expected tags are complete.
- **`blowgun` (a weapon) has `stack: 25` and an `ammunition` tag** — weapons shouldn't
  stack; it's been tagged like ammo. `dart` (stack 10) and `javelin` (stack 5) are thrown
  weapons where a small stack is arguably fine, but `blowgun` is a held weapon.

### weight / stack sanity
- `bone-dust` and `parchment` weigh 0 (non-set items that should probably weigh a token
  amount). The 10 Armor Sets at weight 0 are intentional (virtual bundles).
- `gold-piece` stack = 1e12 sentinel (§3.11).
- Thrown/ammo stacking is inconsistent: `dart` 10, `javelin` 5, `blowgun` 25,
  `sling-bullet`/`arrows` in the ammo type — no single rule.

## 7. Recommendations (prioritized for the item-refiner)

**Tier 1 — schema normalization (mechanical, low-risk, do first):**
1. Delete the dead `img` field from all 131 items (it's a stale category icon, never a
   sprite path).
2. Fix the `substitues`→`substitutes` typo and decide the key's fate (populate or remove;
   they're all empty strings today). Apply one convention to Arcane + Druidic + Holy foci.
3. Remove the singleton redundant keys: `pouch.contains`, `pouch.slots`, `quiver.container`
   (and reconcile `pouch`'s conflicting 4-vs-20 slot count → keep `container_slots: 20`).
4. Normalize field-name casing: `damage-type`→`damage_type`, `range-long`→`range_long`
   (confirm reader keys first). Normalize the value `bludgeon`→`bludgeoning`.
5. Make `allowed_types` always a list; fix `spellbook`'s bogus `"spell-scroll"` type.

**Tier 2 — reference fixes (correctness):**
6. Fix broken weapon `ammunition`: `bolts`→`crossbow-bolts`, `sling bullets`→`sling-bullet`;
   remove `ammunition`/`arrows` from thrown weapons (`dart`, `net`); re-model `net` as a
   thrown restraint (its `damage: "0"` + `range-long: "320"` are wrong).
7. Add `"set": "splint-set"` to `splint-greaves` (close the one asymmetric armor link).
8. Rename the 10 Armor-Set sprites (or the `image` paths) so `{id}.png` holds — and
   generate the 28 missing sprites (Packs, healing Potions, set bundles).
9. Fix `dart`'s `gear_slot: "ammo"` (it's a held thrown weapon) and audit `gear_slot:
   "hands"` (33 items) against the mainhand/offhand equipment model.

**Tier 3 — content + balance (judgment):**
10. Price the 37 zero-value weapons (they're free in shops right now) — the highest-impact
    balance fix.
11. Reconcile rarity↔value tiers for armor (`breastplate` common@40k vs
    `halfplate-cuirass` rare@45k) and fix `case-map-and-scroll` (rare@1).
12. Write the 63 `"NEEDS DESCRIPTION"` descriptions.
13. Coordinate with the **spell-refiner**: 11 Spell Components are unused by any spell
    (several are focus-provided) — either add spells that need them or trim.
14. Backfill tags on the 59 empty-`tags` items; drop `blowgun`'s stack/ammunition tagging.
